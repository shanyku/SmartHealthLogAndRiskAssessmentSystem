# 智慧健康日誌與風險評估系統 (Smart Health Log & Risk Assessment System)

本專案為期末挑戰任務——題目 A：智慧健康日誌與風險評估系統。這是一個具備實戰價值的全端系統，使用者可以記錄每日的健康數據（睡眠時數、步數、心情分數），系統會透過後端嵌入的「決策樹分類模型」自動評估目前的健康風險等級（低 / 中 / 高），並即時呈現在前端介面上。

## 🛠️ 核心考核核心：技術架構與資料流 (Data Flow)

本系統完全符合「Frontend → API → DB」的清晰資料流架構，確保資料在各層級間流暢且正確地連動：

1. **前端 (Frontend - 使用者介面)**
   - 提供直覺的「輸入表單」，供使用者填寫今日的睡眠時數、當日步數及心情分數（1~10 分）。
   - 具備「歷史紀錄列表」，從後端 API 撈取過去的所有健康日誌，並以清晰的表格或圖表呈現數據趨勢。
   - 包含「風險等級徽章」，利用顏色（綠色：低風險 / 黃色：中風險 / 紅色：高風險）即時呈現評估結果。

2. **後端 (Backend API - 商務邏輯與模型整合)**
   - 使用 **Node.js** 與 **Express** 框架建立輕量級 RESTful API 伺服器。
   - 嵌入核心「決策樹邏輯（Decision Tree Classifer）」，當收到前端請求或建立新紀錄時，依序判斷睡眠、步數、心情特徵，動態分類風險等級。
   - 提供完整的 API 端點供前後端串接。

3. **資料庫 (Database - 數據持久化)**
   - 採用 **SQLite** 嵌入式資料庫，無須額外安裝繁重的資料庫軟體，直接以單一檔案存儲。
   - 設計 `health_logs` 資料表，妥善保存歷史健康數據與模型分類後的風險結果。

## 📐 決策樹模型與應用整合 (DM/ML Core)

本系統的核心亮點在於**將機器學習/資料探勘的決策樹分類結果真正嵌入商務邏輯中**：

### 1. 決策樹判斷架構
系統並非使用單一的 `if` 條件，而是實作了具備多層分支的決策樹結構：
- **第一層：睡眠時數判斷**
  - 當睡眠少於 5.5 小時，進入高風險群潛在判斷。
- **第二層：當日步數判斷**
  - 在睡眠不足的情況下，若步數亦少於 3500 步，則進一步深入分析。
- **第三層：心情分數判斷**
  - 綜合最後的心情分數（1~10分），最終切分出 **低風險 (Low)**、**中風險 (Medium)**、**高風險 (High)** 三種標籤。

### 2. 資料訊號與種子資料驗證
- 系統透過腳本（`initDB.js`）成功匯入了 **90 天的健康日誌種子資料**。
- 資料分布經過刻意安排，具備明顯的特徵訊號（如：高風險組合、普通混合組、健康足量組），避免數據純隨機化，確保決策樹模型能切分出具備統計與實務意義的結果。

## 📡 建議與實作之 API 端點 (API Endpoints)

| 方法 (Method) | 路徑 (Path) | 用途 (Description) |
| :--- | :--- | :--- |
| **GET** | `/health-logs` | 取得資料庫中所有的健康日誌紀錄（供前端列表顯示） |
| **POST** | `/health-logs` | 新增一筆健康日誌（送出時由後端決策樹計算風險等級並寫入 DB） |
| **PUT** | `/health-logs/:id` | 修改指定 ID 的健康日誌數據 |
| **DELETE**| `/health-logs/:id` | 刪除指定 ID 的健康日誌紀錄 |
| **GET** | `/health-logs/risk` | 依目前的最新數據，由決策樹邏輯計算並即時回傳風險等級 |
